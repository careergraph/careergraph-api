# V2.1 - Auto-sync Candidate ES when CV Changes

## Overview

Hệ thống tự động đồng bộ candidate vào Elasticsearch khi CV thay đổi:

1. **CV upload** → Auto extract & sync
2. **CV update** (shareToFindJob toggle, text extracted) → Re-sync  
3. **CV delete** → Re-sync để remove CV khỏi ES

## Architecture

```
File CRUD Operation
  ↓
Publish FileChangedEvent
  ↓  
FileEventListener (async, after transaction commit)
  ↓
CandidateESService.syncCandidate()
  ↓
Fetch ALL CVs với shareToFindJob = true
  ↓
Aggregate CV texts + keywords → Index to ES
```

## Implementation

### 1. Publish FileChangedEvent

Trong FileService/Controller, sau khi save/update/delete File:

```java
@Service
@RequiredArgsConstructor
public class FileServiceImpl {
  
  private final ApplicationEventPublisher eventPublisher;
  private final FileRepository fileRepository;
  
  public File uploadResume(File file) {
    // Save file
    File savedFile = fileRepository.save(file);
    
    // Publish event để trigger ES sync (CHỈ với CANDIDATE CV/RESUME)
    if (shouldPublishEvent(savedFile)) {
      eventPublisher.publishEvent(new FileChangedEvent(
        this,
        savedFile.getOwnerId(),      // candidateId  
        savedFile.getId(),             // fileId
        savedFile.getFileName(),       // fileName
        FileChangedEvent.ChangeType.CREATED
      ));
    }
    
    return savedFile;
  }
  
  public File updateFile(String fileId, File updates) {
    File file = fileRepository.findById(fileId)
        .orElseThrow(() -> new RuntimeException("File not found"));
    
    // Update fields
    file.setShareToFindJob(updates.getShareToFindJob());
    // ... other updates
    
    File updatedFile = fileRepository.save(file);
    
    // Publish event
    if (shouldPublishEvent(updatedFile)) {
      eventPublisher.publishEvent(new FileChangedEvent(
        this,
        updatedFile.getOwnerId(),
        updatedFile.getId(),
        updatedFile.getFileName(),
        FileChangedEvent.ChangeType.UPDATED
      ));
    }
    
    return updatedFile;
  }
  
  public void deleteFile(String fileId) {
    File file = fileRepository.findById(fileId)
        .orElseThrow(() -> new RuntimeException("File not found"));
    
    String candidateId = file.getOwnerId();
    String fileName = file.getFileName();
    
    fileRepository.delete(file);
    
    // Publish event
    if (shouldPublishEvent(file)) {
      eventPublisher.publishEvent(new FileChangedEvent(
        this,
        candidateId,
        fileId,
        fileName,
        FileChangedEvent.ChangeType.DELETED
      ));
    }
  }
  
  private boolean shouldPublishEvent(File file) {
    if (file == null || file.getOwnerId() == null) {
      return false;
    }
    
    // Chỉ publish event cho CANDIDATE CV/RESUME
    return file.getOwnerType() == OwnerType.CANDIDATE &&
           (file.getFileType() == FileType.RESUME || file.getFileType() == FileType.CV);
  }
}
```

### 2. CV Extraction Service

Khi CV được extract (text/keywords), cũng cần publish event:

```java
@Service
public class CvExtractionService {
  
  private final FileRepository fileRepository;
  private final ApplicationEventPublisher eventPublisher;
  
  public void onCvExtracted(String fileId, String extractedText, String keywords) {
    File file = fileRepository.findById(fileId)
        .orElseThrow(() -> new RuntimeException("File not found"));
    
    // Update extracted data
    file.setResumeExtractedText(extractedText);
    file.setCvKeywordsJson(keywords);
    
    File updated = fileRepository.save(file);
    
    // Trigger ES sync
    if (updated.getOwnerType() == OwnerType.CANDIDATE) {
      eventPublisher.publishEvent(new FileChangedEvent(
        this,
        updated.getOwnerId(),
        updated.getId(),
        updated.getFileName(),
        FileChangedEvent.ChangeType.UPDATED
      ));
    }
  }
}
```

### 3. Toggle shareToFindJob

Khi user toggle shareToFindJob:

```java
@RestController
public class CandidateController {
  
  private final FileRepository fileRepository;
  private final ApplicationEventPublisher eventPublisher;
  
  @PatchMapping("/api/v1/candidates/{candidateId}/resume/{fileId}/share-to-find-job")
  public ResponseEntity<?> toggleShareToFindJob(
      @PathVariable String candidateId,
      @PathVariable String fileId,
      @RequestBody Map<String, Boolean> payload) {
    
    Boolean shareToFindJob = payload.get("shareToFindJob");
    
    File file = fileRepository.findById(fileId)
        .orElseThrow(() -> new RuntimeException("File not found"));
    
    file.setShareToFindJob(shareToFindJob);
    File updated = fileRepository.save(file);
    
    // Trigger ES sync để update/remove CV
    eventPublisher.publishEvent(new FileChangedEvent(
      this,
      candidateId,
      fileId,
      updated.getFileName(),
      FileChangedEvent.ChangeType.UPDATED
    ));
    
    return ResponseEntity.ok(updated);
  }
}
```

## Flow

### Case 1: CV Upload

```
1. User upload CV → FileController.upload()
2. File saved → fileRepository.save()
3. Publish FileChangedEvent(CREATED)
4. Background: CV extraction service extract text/keywords
5. Extraction done → fileRepository.save(updated)
6. Publish FileChangedEvent(UPDATED)
7. FileEventListener → syncCandidate()
8. Fetch ALL CV với shareToFindJob=true → Index to ES
```

### Case 2: Toggle shareToFindJob

```
1. User click "Share CV to HR"
2. API call PATCH /candidates/{id}/resume/{fileId}/share-to-find-job
3. Update file.shareToFindJob = true
4. Publish FileChangedEvent(UPDATED)
5. FileEventListener → syncCandidate()
6. Fetch CVs → Now includes this CV → Index to ES
```

### Case 3: CV Delete

```
1. User delete CV
2. FileService.delete(fileId)
3. Publish FileChangedEvent(DELETED)
4. FileEventListener → syncCandidate()
5. Fetch CVs → This CV excluded → Index to ES (CV removed)
```

## Notes

- **Async execution**: FileEventListener chạy async AFTER transaction commit → không block main flow
- **Multiple CVs**: Aggregate ALL CVs có shareToFindJob=true, mỗi CV limit 1000 chars
- **Error handling**: Nếu sync fail, log error nhưng không throw exception (async nên không fail main transaction)
- **Performance**: Dùng @Async annotation → cần enable trong Application config:

```java
@SpringBootApplication
@EnableAsync  // Enable async support
public class CareerGraphApplication {
  // ...
}
```
