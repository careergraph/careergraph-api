## 🧪 Test Scenario 1: Manual Sync Candidate

**Goal:** Verify candidate được sync ngay vào Elasticsearch

### Step 1: Get Candidate ID

```bash
# Candidate vừa tạo: Thịnh Lương
CANDIDATE_ID="837ebdc4-721a-4133-b511-481b03d951fd"
```

### Step 2: Trigger Manual Sync

```bash
# Get auth token first (login as HR/Company)
# Replace <YOUR_TOKEN> với token thực tế

curl -X POST "http://localhost:8010/careergraph/api/v1/candidates/suggestion/sync/${CANDIDATE_ID}" \
  -H "Authorization: Bearer <YOUR_TOKEN>" \
  -H "Content-Type: application/json"
```

**Expected Response:**
```json
{
  "status": "OK",
  "message": "Successfully synced candidate 837ebdc4-721a-4133-b511-481b03d951fd to Elasticsearch",
  "data": "837ebdc4-721a-4133-b511-481b03d951fd"
}
```

### Step 3: Verify in Elasticsearch

```bash
# Check if candidate exists in ES index
curl "http://localhost:9200/candidates_es/_search" \
  -H "Content-Type: application/json" \
  -d "{
    \"query\": {
      \"term\": { \"id\": \"${CANDIDATE_ID}\" }
    }
  }"
```

**Expected Response:**
```json
{
  "hits": {
    "total": { "value": 1 },
    "hits": [{
      "_source": {
        "id": "837ebdc4-721a-4133-b511-481b03d951fd",
        "fullName": "Thịnh Lương",
        "desiredPosition": "Java Developer",
        "cvKeywords": "Java Spring Boot MySQL...",
        "isOpenToWork": true
      }
    }]
  }
}
```

**Status:** ✅ If found, ❌ if not found

## 🧪 Test Scenario 3: Loading States UX

**Goal:** Verify loading feedback rõ ràng

### Step 1: Search With Keyword

1. Trang HR Candidate Search
2. Type keyword: "Java Developer"

### Step 2: Observe Loading State

**Timing:**
- T=0ms: User types "J"
- T=100ms: Debounce timer running
- T=500ms: API call triggers

**Expected Behavior:**

**Phase 1: Typing (0-500ms)**
```
✅ Spinner appears immediately
✅ Text: "Đang tìm kiếm ứng viên..."
✅ Old results stay visible với opacity 60%
✅ Pagination disabled (pointer-events-none)
```

**Phase 2: Loading (500-1000ms)**
```
✅ Spinner still visible
✅ API request in flight
✅ Old results still visible (dimmed)
```

**Phase 3: Results Loaded (1000ms+)**
```
✅ Spinner disappears
✅ New results appear
✅ Opacity back to 100%
✅ Smooth transition (no jarring)
```

**Before Fix:**
```
❌ No spinner during debounce
❌ Old results disappear suddenly
❌ New results appear suddenly (jarring)
```

**Status:** ✅ Smooth UX, ❌ Jerky/confusing

## 📊 Checklist Summary

Use this checklist to verify all fixes:

### Backend Fixes

- [ ] **Manual sync endpoint exists**
  - `POST /api/v1/candidates/suggestion/sync/{candidateId}` returns 200
  
- [ ] **Sync works correctly**
  - Candidate appears in Elasticsearch after sync
  - Query ES index confirms candidate data
  
- [ ] **No regression**
  - Existing search still works
  - Scheduled sync still runs (check logs every 4 min)

### Frontend Fixes

- [ ] **Pagination displays all candidates**
  - If backend returns 10 → Frontend shows 10 (or paginated correctly)
  - Total count matches backend
  - Page navigation works (prev/next arrows)
  
- [ ] **Loading states visible**
  - Spinner appears during search
  - "Đang tìm kiếm ứng viên..." text visible
  - Old results stay visible with opacity during load
  
- [ ] **Smooth UX**
  - No jarring transitions
  - No sudden disappearance of results
  - Clear feedback at all times

### Integration Tests

- [ ] **End-to-end flow**
  - Upload CV → Extract → Sync → Search → Found
  - Total time < 30 seconds (vs 4 minutes before)
  
- [ ] **Multiple searches**
  - Keyword search works ("Java Developer")
  - Blank keyword search works (matches company jobs)
  - Filter by location works
  
- [ ] **Edge cases**
  - Zero results → Show "Không tìm thấy ứng viên"
  - 1000+ results → Pagination works correctly
  - Network slow (simulate 3G) → Loading state visible

## 🐛 Known Issues (If Any)

### Issue 1: Token Expiration

**Symptom:** Sync endpoint returns 401 Unauthorized

**Solution:**
```bash
# Re-login to get fresh token
# Or use Postman with "Send and request new token"
```

### Issue 2: Elasticsearch Not Running

**Symptom:** Sync returns 500, logs show "Connection refused localhost:9200"

**Solution:**
```bash
# Start Elasticsearch
docker-compose up -d elasticsearch

# Verify
curl http://localhost:9200/_cluster/health
```

### Issue 3: Frontend Shows Old Candidates

**Symptom:** Sync successful nhưng frontend chưa update

**Solution:**
```bash
# Hard refresh browser
Ctrl + Shift + R (hoặc Cmd + Shift + R on Mac)

# Clear browser cache
# Or use Incognito mode
```
